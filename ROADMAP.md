# Roadmap: 20 фитч для лидерства на рынке

> Цель: сделать labus продуктом, который ресторан подключает за 15 минут,
> гость открывает с удовольствием, а владелец не хочет менять ни на что другое.

---

## Для гостей ресторана

### 1. Умные рекомендации «Вам понравится»

Блок в верхней части меню, персонализированный под конкретного гостя.
- Новый гость: показываем топ-3 по продажам за неделю (социальное доказательство)
- Повторный гость: анализ истории заказов, предпочтений по категориям, среднего чека
- Контекст: время суток (завтрак/обед/ужин), день недели
- Метка «Вы заказывали» на позициях из прошлых визитов

**Эффект для ресторана:** +12-17% к среднему чеку. Гость не листает всё меню — он видит релевантное сразу.

---

### 2. Допродажа в корзине

При добавлении блюда в корзину — всплывающее предложение сопутствующих позиций.
- Связки настраиваются рестораном: пицца → напиток, бургер → картофель, десерт → кофе
- Если не настроено — автоматические связки по истории совместных заказов
- Ненавязчиво: одна карточка под кнопкой «В корзину», закрывается свайпом

**Эффект:** +8-15% к среднему чеку без раздражения гостя.

---

### 3. Программа лояльности с кэшбеком

Встроенная система баллов, работающая без пластиковых карт и сторонних приложений.
- Начисление: X% от суммы заказа возвращается баллами (ресторан задаёт процент)
- Списание: баллами можно оплатить до Y% следующего заказа
- Приветственный бонус при первом заказе
- Бонус за день рождения (гость указывает дату в профиле)
- Баланс виден в аккаунте и в корзине при оформлении

**Эффект:** удержание. Гость возвращается, потому что у него «лежат» баллы.

---

### 4. Отзывы и рейтинг блюд

После завершения заказа (статус «завершён» + 30 минут) — push с просьбой оценить.
- Оценка каждого блюда: 1-5 звёзд + опциональный текст
- Рейтинг отображается в меню рядом с названием (средняя оценка + кол-во отзывов)
- Негативные отзывы (1-2 звезды) не публикуются, а приходят владельцу в Telegram как сигнал
- Модерация: ресторан может ответить на отзыв

**Эффект:** социальное доказательство помогает новым гостям выбрать, а ресторану — понять проблемы.

---

### 5. Модификаторы и кастомизация блюд

Возможность настроить блюдо при заказе.
- Размер/объём (маленький, средний, большой) — разная цена
- Добавки за доплату (сыр +50₽, бекон +80₽)
- Исключения бесплатно (без лука, без глютена)
- Обязательные выборы (степень прожарки стейка, тип молока в кофе)
- Настраивается рестораном в админке для каждого блюда

**Эффект:** ресторан перестаёт терять заказы от гостей с аллергиями/предпочтениями.

---

### 6. Живой трекинг заказа

Визуальный прогресс-бар заказа в реальном времени — как в Яндекс.Еде.
- 4 шага: Принят → Готовим → Доставляем/Готов к выдаче → Завершён
- Анимация перехода между этапами
- Push-уведомление на каждый переход статуса
- Примерное время готовности (среднее по категории блюд из истории)

**Эффект:** снижает тревожность гостя и кол-во звонков «где мой заказ?».

---

### 7. Оплата онлайн

Интеграция с платёжным шлюзом (ЮKassa / Тинькофф / СБП).
- Оплата при оформлении заказа (до отправки на кухню)
- Оплата по ссылке (официант отправляет счёт в чат)
- Чаевые: кнопки 5%/10%/15%/другая сумма при оплате
- Разделение счёта между гостями (каждый оплачивает свою часть)

**Эффект:** ускоряет оборот стола, снижает нагрузку на кассу, чаевые повышают лояльность персонала.

---

### 8. Мультиязычность меню

Переключатель языка в шапке меню.
- Ресторан заполняет названия/описания на нескольких языках в админке
- Если перевода нет — показывается оригинал
- Язык по умолчанию: русский. Дополнительные: английский, арабский, китайский, турецкий
- Язык сохраняется в профиле гостя

**Эффект:** критично для ресторанов в туристических зонах. Конкуренты в РФ этого почти не делают.

---

## Для ресторана (владелец/сотрудники)

### 9. Стоп-лист в реальном времени

Моментальное скрытие позиции из меню когда она закончилась.
- Кнопка «СТОП» напротив каждого блюда в админке/у сотрудника
- Блюдо становится серым с пометкой «Нет в наличии» (не исчезает, чтобы гость знал что оно существует)
- Опциональный авто-стоп: привязка к остаткам ингредиентов (если ведётся складской учёт)
- Push-уведомление владельцу когда блюдо попадает в стоп

**Эффект:** ноль ситуаций «заказал, а потом сказали что нет».

---

### 10. QR-привязка к столу (сессия стола)

Уникальный QR на каждом столе, при сканировании — автоматическая привязка к номеру стола.
- Генерация QR-кодов для печати из админки (PDF, один лист на стол)
- Гость сканирует → меню открывается с привязкой к столу (не надо вводить номер)
- Все гости за одним столом видят общий заказ (по session QR)
- Дозаказ без пересканирования: кнопка «Добавить к заказу» на том же столе
- Официант видит: стол №5 → 3 заказа → итого 2400₽

**Эффект:** убирает путаницу «какой стол?», ускоряет обслуживание.

---

### 11. Кухонный экран (KDS)

Отдельная страница `/kitchen.php` для планшета на кухне.
- Карточки заказов в реальном времени (SSE)
- Разбивка по блюдам, не по заказам (повар видит «3 пиццы Маргарита» а не три отдельных заказа)
- Кнопка «Готово» на каждом блюде → когда все блюда заказа готовы → автосмена статуса
- Таймер на каждом заказе (сколько минут с момента принятия)
- Цветовая индикация: зелёный (<10мин), жёлтый (10-20мин), красный (>20мин)

**Эффект:** замена бумажных чеков на кухне. Прозрачность для владельца — видно где тормозит.

---

### 12. Уведомления по каналам: Telegram-бот для персонала

Расширение текущего Telegram-бота в полноценный инструмент управления.
- Бот для официантов: новый заказ → кнопки «Принять / Отклонить» прямо в Telegram
- Бот для повара: список активных заказов, кнопка «Готово»
- Бот для владельца: дневная сводка (выручка, кол-во заказов, средний чек, топ-3 блюда)
- Настройка: кто какие уведомления получает (по ролям)

**Эффект:** персонал управляет заказами без отдельного приложения — через Telegram, который уже у всех.

---

### 13. Аналитика по блюдам: ABC-анализ

Автоматическая категоризация блюд по прибыльности и популярности.
- **A** — звёзды: высокая маржа + высокие продажи (продвигать)
- **B** — рабочие лошадки: средние показатели (оптимизировать)
- **C** — балласт: низкая маржа + низкие продажи (убрать или переработать)
- Визуализация: матрица 2x2 (маржа × объём продаж)
- Рекомендации: «Салат Цезарь приносит 2% выручки и 0.5% прибыли — рассмотрите замену»

**Эффект:** владелец принимает решения на данных, а не на интуиции.

---

### 14. Управление несколькими заведениями

Одна учётная запись владельца → несколько ресторанов.
- Переключение между ресторанами в шапке админки
- Общая аналитика по всем точкам + отдельная по каждой
- Копирование меню из одного заведения в другое
- Единая программа лояльности (баллы работают во всех точках сети)
- Роли сотрудников привязаны к конкретному заведению

**Эффект:** ключевой фактор для сетей. Без этого — потолок в 1 ресторан на клиента.

---

### 15. Расписание меню

Разные меню для разного времени суток.
- Завтрак (08:00-11:00), Обед (11:00-16:00), Ужин (16:00-23:00), Ночное (23:00-08:00)
- Категории/блюда привязываются к расписанию
- Автопереключение — гость в 9 утра видит завтраки, в 13:00 — бизнес-ланч

**Эффект:** ресторан не путает гостей утренними позициями вечером и наоборот.

---

### 16. Промо-баннеры и акции

Возможность добавить визуальные акции в меню.
- Баннер-карусель вверху меню (настраивается в админке: изображение, текст, ссылка на категорию/блюдо)
- Блюдо дня / Комбо-предложение с перечёркнутой старой ценой
- Таймер акции (например, «Счастливые часы: ещё 2ч 15мин»)
- Автоскрытие по дате окончания

**Эффект:** инструмент для продвижения конкретных позиций без изменения всего меню.

---

## Платформенные (масштабирование продукта)

### 17. Белая метка (White Label) за 15 минут

Полная кастомизация вида под бренд ресторана без написания кода.
- Визуальный конструктор: загрузка лого, выбор цветовой схемы, шрифтов (уже есть основа)
- Собственный домен: ресторан подключает свой домен (menu.myrestaurant.ru), SSL автоматически
- Кастомный favicon и PWA-иконки
- Удаление упоминаний labus — ресторан видит только свой бренд
- Кастомные тексты: приветствие, описание, ссылки на соцсети

**Эффект:** ресторан получает «своё приложение» без разработки. Главный аргумент при продаже.

---

### 18. Автоонбординг: демо-меню + пошаговый визард

Ресторан регистрируется и за 15 минут получает рабочее меню.
- Шаг 1: регистрация (email + название ресторана)
- Шаг 2: выбор типа кухни → подгружается демо-меню (итальянская, японская, кофейня, бар и т.д.)
- Шаг 3: загрузка логотипа, выбор цветов
- Шаг 4: генерация QR-кода для первого стола
- Шаг 5: меню работает, ресторан редактирует под себя
- Демо-данные: 15-20 позиций с фото, описаниями, ценами-заглушками

**Эффект:** нулевой порог входа. Ресторан видит результат ДО того как заплатил.

---

### 19. Витрина ресторанов (маркетплейс)

Единая страница-каталог всех подключённых ресторанов.
- Карточка ресторана: лого, название, тип кухни, рейтинг, адрес, ссылка на меню
- Фильтры: тип кухни, рейтинг, доставка/самовывоз, геолокация (рядом со мной)
- SEO: каждый ресторан — отдельная индексируемая страница
- Опционально для ресторана: можно не участвовать в витрине (только прямые ссылки)

**Эффект:** органический трафик. Гость ищет «где поесть» и находит рестораны с labus.

---

### 20. Панель тарифов и биллинг

Монетизация платформы через подписку.
- **Бесплатный:** меню + QR + 50 заказов/мес
- **Стандарт:** +доставка, +лояльность, +аналитика, +безлимит заказов
- **Премиум:** +кухонный экран, +мультиточки, +white label, +приоритетная поддержка
- Личный кабинет ресторана: текущий тариф, лимиты, история платежей
- Автопродление через ЮKassa/Stripe
- Триал: 14 дней полного функционала бесплатно

**Эффект:** масштабируемая бизнес-модель. Ресторан платит за ценность, не за «кота в мешке».

---

### 21. Интеграция с iiko (iikoCloud API)

iiko (айко) — самая распространённая система автоматизации ресторанов в России (29 000+ заведений).
Интеграция позволяет использовать cleanmenu как красивый фронтенд поверх уже работающей учётной системы — без двойного ввода данных.

**Что даёт интеграция:**
- **Синхронизация меню:** блюда, категории, описания, фото, цены и модификаторы загружаются из iiko автоматически — через номенклатуру или внешнее меню iikoCloud
- **Стоп-лист в реальном времени:** если блюдо заканчивается на кухне, оно автоматически скрывается в меню через webhook или polling к iiko API
- **Передача заказов:** заказ, оформленный через cleanmenu, создаётся в iiko как столичный заказ — кассир и кухня видят его в своих экранах без ручного ввода
- **Синхронизация цен:** изменение цены в iiko → моментальное обновление в cleanmenu
- **Модификаторы:** группы модификаторов (размер, добавки, соусы) тянутся из iiko и отображаются гостю при выборе блюда
- **Статус заказа:** cleanmenu получает статус приготовления из iiko и обновляет трекер заказа (готовим → готов)

**Техническая реализация:**
- Используется iikoCloud API (REST, ключ генерируется в личном кабинете iiko)
- Поддерживаемые версии: iikoCloud и iikoClassic 7.9.7+
- Настройка: в панели администратора вводится API-ключ iikoCloud и ID терминальной группы
- Синхронизация меню: по кнопке «Обновить из iiko» или по расписанию (cron)
- Передача заказов: через `POST /api/1/deliveries/create` с маппингом блюд cleanmenu → iiko nomenclatureId

**Эффект:** ресторан, уже работающий на iiko, подключается за 30 минут. Не нужно вести два меню и вручную принимать заказы из электронного меню.
