## Major (7)

### M1 · `js/file-manager.min.js` 404 on `/admin/menu.php` ✅ FIXED
**Symptom:** Console shows `GET /admin/file-manager.php?action=get_fonts → 404`.
**Root cause:** `js/file-manager.min.js` uses relative URLs (5 fetch'ей). After Phase 13B.3.a moved `admin-menu.php → admin/menu.php`, the relative paths resolve under `/admin/` instead of root.
**Fix:** prefix `/` to 5 fetch URLs (L95, L116, L166, L231, L556).

### M2 · `/admin/menu.php?tab=dishes` ignores URL param ✅ FIXED
**Symptom:** Loading `/admin/menu.php?tab=dishes` defaults to «Система» tab.
**Root cause:** `js/admin-tabs-repair.js` reads localStorage but never `URLSearchParams.get('tab')`.
**Fix:** insert URL-param check between localStorage read and click-handlers wiring; activate matching tab and persist to localStorage.

### M3 · `account.php?tab=updates` shows fake "Critical update" modal ✅ FIXED
**Symptom:** Modal triggers on every visit despite app already at 2.0.0.
**Root cause:** `closeModal()` in `js/version-checker.min.js` only persisted `last_version_check` timestamp — never the dismissed `app_version`.
**Fix:** add `localStorage.setItem('app_version', newVersion)` in `closeModal(true)` branch.

### M4 · `\r\n` literal in changelog (Phase 10.7 regression) ✅ FIXED
**Symptom:** End of changelog shows literal `` `r`n `` chars on `/account.php?tab=updates`.
**Root cause:** PowerShell heredoc artifact left in `account.php:409` from a past nano edit.
**Fix:** remove the two stray characters between `<div>` and `<a href>`.

### M5 · Page hero headings clipped under sticky nav ✅ FIXED
**Surfaces:** `/reservation.php`, `/group.php`, `/signup.php`.
**Root cause:** these surfaces don't include `account-header.php` (which provides the `#menu` element receiving 96px padding-top). No body-level fallback existed.
**Fix:** add `body.reservation-page, body.group-page, body.signup-page { padding-top: 96px; }` to `css/account-styles.min.css` (loaded by all three).

### M6 · `/menu.php` — 7 broken product images on Test catalog ✅ FIXED (script ready)
**Symptom:** 7×404 on `/images/Pizza/{piperoni,margarita,misnaya}.jpg`, `/images/Sets/mangal.jpg`, `/images/Snaks/{meatnut,meat,piperonichips}.jpg`.
**Root cause:** test menu_items created manually via UI on tab «Тест», pointing to non-existent files.
**Fix:** new `scripts/cleanup-broken-image-items.php` archives the 7 rows by `image` path. Run once on prod after deploy.

### M7 · Homepage «О сервисе» — empty pink rectangle (Phase 9.9 regression) ⏳ DEPLOY-VERIFY
**Symptom:** Right column shows blank pink-tinted rectangle where HDR1 webp image should be.
**Status:** files `images/HDR1_{320,640,1024}.webp` exist locally. Markup in `index.php:149-167` correct.
**Action on deploy:** `curl -I https://menu.labus.pro/images/HDR1_1024.webp`. If 404 — assets missing on prod, push commit. If 200 — investigate CSP `img-src` or MIME handling separately.
